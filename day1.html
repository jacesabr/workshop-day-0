🔄 The Big Shift

📌 Today's app: a simple to-do list. Add tasks, see them in a list.
❌ Old way (meaningless tech jargon):
"I need a MERN stack with MongoDB Atlas and JWT authentication using middleware and a RESTful API with CORS enabled."
✅ AI Revolution way (just describe the screen):
• The home page has a + Add Task button
• When I click it, a white box pops up with a text field and a green Save button
• I type "Buy milk" and click Save
• The box closes, "Buy milk" appears in the list
• Newest task shows at the top
🧠 You never said "database" or "backend". You described a to-do list. The AI figures out the rest.

📢 The One Rule

Don't say tech words. Say what the user sees and what happens when they click.

❌ database ❌ API ❌ backend ❌ endpoint ❌ middleware ❌ schema
✅ "a white box pops up" ✅ "click Save, it appears in the list" ✅ "a green button" ✅ "log in with Google"

2 📝 Describe what happens

Tell Claude what the user sees and what happens when they click — in normal words.

📌 EXAMPLE: Habit Tracker
What it does: Check off habits each day (like "exercise" or "read").

Describe it (no tech words):
  • Main screen: a list of my habits with a checkbox next to each, and today's date at the top
  • "+ New Habit" button at the bottom
  • Click "+ New Habit" → a small box slides up with a text field + blue "Create" button
  • Click Create → box closes, new habit appears in the list
  • Check a checkbox → it turns green
  • Tomorrow → all checkboxes reset
📋 Template — fill in the blanks
"I want an app called [NAME]. What it does: [one sentence] Screens: [list them] Main screen has: [buttons, lists, checkboxes] When I click [BUTTON]: [what happens]"
