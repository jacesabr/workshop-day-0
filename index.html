03Prompt engineering

Prompt engineering.

The same question, asked better. Four small additions to your prompt change the answer dramatically — you’ll re-ask the question from the last section and see for yourself.

01
Tell it who it is

Set the role. The AI takes on different vocabulary, depth, and assumptions depending.

“You are an expert math teacher who explains things to high-schoolers.”
02
Tell it what to do

Be concrete about the action you want. Not just “help me” — “walk me through it”.

“Explain this concept step-by-step. Then quiz me on it.”
03
Give it context

Background changes the answer entirely. Who’s asking? What’s the situation?

“I’m a 12th-grader prepping for boards. I’ve been struggling with this for a week.”
04
Tell it how to answer

Specify format. Bullets vs prose, length, structure, examples.

“Use bullet points. Keep it under 200 words. Show one worked example.”
Your turn
  • Take the complex-thinking question you asked Claude in the last section
  • Re-write it using all four rules above
  • Send it back to Claude — get the new answer
  • Compare: which answer is actually more useful to you?

08How apps work

How apps work.

Every app is the same six pieces. Once you see the parts, you can talk about any of them.

The shape of every app
Frontend
The screen. The buttons, forms, and pages. What the user actually sees and touches.
Backend
The brain. The rules. "When the user taps Submit, do these five things." Lives on a server.
Database
Where stuff is kept. Users, posts, orders, messages — rows in tables you can query.
Storage
Where files go — profile pictures, PDFs, uploads. Different from a database; databases hate big files.
Auth
"Who is this person? Are they allowed to do this?" Sits between frontend and backend.
Hosting
The actual computers, somewhere, running this 24/7 so anyone can reach it.
